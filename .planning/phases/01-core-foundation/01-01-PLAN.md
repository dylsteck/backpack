<task type="auto">
  <name>Define Database Schema</name>
  <files>packages/core/src/db/schema.ts, packages/core/src/types/index.ts</files>
  <action>
Create database schema in schema.ts:

1. timeline_items table:
   - id: text primary key (UUID)
   - source: text not null (farcaster, obsidian, teller, chrome)
   - type: text not null (post, note, transaction, visit, etc.)
   - external_id: text (ID from source system)
   - title: text
   - content: text (main content)
   - raw_data: text (JSON string of full source data)
   - url: text
   - timestamp: integer (Unix ms)
   - created_at: integer (Unix ms)
   - updated_at: integer (Unix ms)
   - sync_status: text (pending, synced, error)
   - error_message: text

2. sources table:
   - id: text primary key
   - name: text not null
   - type: text not null
   - config: text (JSON config)
   - last_sync_at: integer
   - is_enabled: integer (0/1)
   - created_at: integer

3. embeddings table:
   - id: text primary key
   - item_id: text not null (foreign key to timeline_items)
   - vector: blob (binary embedding data)
   - model: text (embedding model used)
   - created_at: integer

4. Create corresponding TypeScript types in types/index.ts:
   - TimelineItem interface matching schema
   - SourceType enum/union
   - SyncStatus enum
   - SourceConfig interfaces for each source type

5. Export everything from packages/core/src/index.ts
  </action>
  <verify>grep -r "timeline_items\|TimelineItem" packages/core/src/ shows schema and types</verify>
  <done>Schema and types defined with full type safety</done>
</task>
