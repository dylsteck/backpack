1. Support multiple concurrent syncs (but throttle to avoid rate limits)
2. Track progress per source
3. Handle partial failures (one source failing shouldn't stop others)
4. Support incremental syncs (only fetch new data)
5. Integrate with auto-embed after sync

   - totalDurationMs: number

5. SyncerInterface (abstract):
   - name: SourceType

   - updateProgress(progress: SyncProgress): void
     * Call onProgress callback if provided

3. Error handling:
   - Wrap doSync in try/catch
   - On error: set status to failed, add error message
   - Still return progress object

This base class ensures all syncers behave consistently.
