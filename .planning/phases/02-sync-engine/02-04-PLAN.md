<task type="auto">
  <name>Implement Obsidian Syncer</name>
  <files>packages/core/src/sync/sources/obsidian.ts</files>
  <action>
Create ObsidianSyncer class extending BaseSyncer:

1. Class structure:
   - name = 'obsidian'
   - config: ObsidianConfig (vaultPath)

2. isConfigured():
   - Check if vaultPath is set in config
   - Verify directory exists
   - Return true/false

3. validateConfig():
   - Check vaultPath exists
   - Verify it's a directory
   - Try to read one file to ensure permissions
   - Return true/false

4. doSync(progress):
   - Walk vaultPath recursively using fs.readdir or glob
   - Filter for .md files only
   - For each file:
     * Get file stats (mtime, size)
     * Check if already synced and unchanged (compare mtime with DB)
     * Read file content
     * Parse frontmatter (using gray-matter or simple regex)
     * Extract wikilinks (regex: \[\[(.*?)\]\])
     * Extract tags (regex: #\w+)
     * Create TimelineItem:
       - id: uuid
       - source: 'obsidian'
       - type: 'note'
       - external_id: relative file path
       - title: filename without .md or frontmatter.title
       - content: markdown body
       - raw_data: JSON.stringify({ frontmatter, wikilinks, tags, path })
       - url: file://path
       - timestamp: frontmatter.created or file birthtime
       - created_at: now
       - updated_at: file mtime
     * Save via this.saveItem()
     * Update progress counters
   
   - Handle errors per-file (log and continue)
   - Return updated progress

5. Helper methods:
   - parseFrontmatter(content): returns { data, body }
   - extractWikilinks(content): string[]
   - extractTags(content): string[]
   - getFileHash(path): for change detection

6. Incremental sync:
   - Store file mtime in raw_data
   - On next sync, skip files where mtime matches
   - This makes subsequent syncs very fast

Export ObsidianSyncer class.
  </action>
