<task type="auto">
  <name>Implement Farcaster Syncer</name>
  <files>packages/core/src/sync/sources/farcaster.ts</files>
  <action>
Create FarcasterSyncer class:

1. Class structure:
   - name = 'farcaster'
   - config: FarcasterConfig
   - neynarApiKey?: string (from config or env)

2. isConfigured():
   - Check if fid is set
   - Check if signer is in keychain (getSecret('farcaster-signer'))
   - If using Neynar, check apiKey exists
   - Return true/false

3. validateConfig():
   - Test API call with small limit (1 cast)
   - Verify authentication works
   - Return true/false

4. doSync(progress):
   - Determine API to use (Neynar vs direct hub)
   - Get last cursor from previous sync (stored in source config or query last item)
   - Fetch casts in batches:
     * Neynar: GET /v2/farcaster/feed/user/fid
     * Params: limit (100), cursor (from last sync)
   - For each cast:
     * Create TimelineItem:
       - id: uuid
       - source: 'farcaster'
       - type: 'cast'
       - external_id: cast hash
       - title: null (casts don't have titles)
       - content: cast text
       - raw_data: JSON.stringify(full cast object)
       - url: https://warpcast.com/{username}/{hash}
       - timestamp: new Date(cast.timestamp).getTime()
       - created_at: now
       - updated_at: now
     * Save via this.saveItem()
   - Handle pagination:
     * Continue until no more casts or cursor
     * Store cursor for next incremental sync
   - Rate limiting:
     * Add delay between requests (100ms for 10 req/sec)
   - Update progress counters
   - Return progress

5. API client methods:
   - async fetchNeynarCasts(fid: number, cursor?: string): Promise<Cast[]>
   - async fetchHubCasts(fid: number): Promise<Cast[]> (alternative)

6. Error handling:
   - Handle 429 rate limit (backoff and retry)
   - Handle auth errors (401/403)
   - Log specific error messages

Export FarcasterSyncer.
