  <name>Implement Embedding Generation</name>
  <files>packages/core/src/embeddings/index.ts</files>
  <action>
Create embedding orchestration:

1. Text extraction function:
   - extractEmbeddingText(item: TimelineItem): string
   - Different extraction per source:
     * obsidian: `${item.title} ${item.content}`
     * farcaster: item.content (cast text)
     * teller: `${item.title} ${JSON.parse(item.raw_data).transaction.description}`
     * chrome: `${item.title} ${item.url}`
   - Fallback: item.content || item.title

2. Generate embeddings for items:
   - async generateEmbeddings(itemIds: string[]): Promise<void>
     * Fetch items from database by IDs
     * Filter out items without content
     * Extract text for each
     * Batch into chunks (100 items at a time)
     * Call qmdClient.indexItems() for each batch
     * Update items in DB to mark as embedded (add embedded_at timestamp)
     * Log progress

3. Auto-generate after sync:
   - async generateForNewItems(): Promise<void>
     * Query DB for items where embedded_at IS NULL
     * If items found:
       * Log: "Generating embeddings for N new items..."
       * Call generateEmbeddings(itemIds)
       * Log: "Embeddings complete"
     * If no items:
       * Log: "No new items to embed"

4. Integration with sync manager:
   - Modify syncManager.syncAll() to call generateForNewItems() after successful sync
   - Make it async and awaited
   - Don't fail sync if embeddings fail (log warning)

5. Re-embedding:
   - async regenerateAllEmbeddings(): Promise<void>
     * For force re-index (if model changes, etc.)
     * Clear existing embeddings in QMD
     * Re-index all items

Export all functions.
